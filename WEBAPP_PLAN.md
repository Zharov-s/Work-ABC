## Суть проекта

Единый веб-интерфейс, объединяющий три текущих проекта:
- **Research** — поиск компаний и ЛПР (лиц, принимающих решения)
- **mitino-email_html** — HTML-рассылка по объекту «Промтехнопарк Митино»
- **grekova_email_html** — HTML-рассылка по объекту «Грекова, 5–7»

**Главная ценность:** из одного браузера — настроить поиск → получить контакты ЛПР → выбрать шаблон письма → отправить рассылку.

**Ключевые ограничения:**
- Рассылка: не более 30 адресов за один раз (лимит Mail.ru BCC)
- Хранилище: SQLite (один файл, никаких внешних БД)
- Запуск: локально (localhost), в будущем возможен перенос на VPS
- Защита: логин/пароль для входа

---

## Технологический стек

| Слой | Технология | Обоснование |
|---|---|---|
| Backend | **Python + Flask** | Уже есть Python-скрипты; Flask минимален и достаточен |
| Frontend | **HTML + Vanilla JS** | Без тяжёлых фреймворков; быстро, понятно |
| База данных | **SQLite** | Один файл, нет сервера БД, легко бэкапить |
| SMTP | **smtplib** (уже есть) | Проверено в обоих email-проектах |
| Research | **subprocess** → существующий `run_accelerated_outreach.py` | Без переписывания логики поиска |
| Стриминг лога | **Server-Sent Events (SSE)** | Показ прогресса поиска в реальном времени |

## Схема базы данных (SQLite)

```sql
-- Контакты ЛПР (импорт из JSONL + все новые)
CREATE TABLE contacts (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    company_name TEXT NOT NULL,
    website      TEXT,
    person_name  TEXT,
    title        TEXT,
    email        TEXT UNIQUE NOT NULL,
    phone        TEXT,
    source_url   TEXT,
    segment      TEXT,          -- электроника, медтех, light industrial, etc.
    region       TEXT,          -- Москва, МО, Россия
    date_found   DATE,
    status       TEXT DEFAULT 'new',  -- new | sent | unsubscribed
    notes        TEXT,
    created_at   TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- История рассылок
CREATE TABLE send_history (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    template        TEXT NOT NULL,   -- mitino | grekova
    subject         TEXT,
    sent_at         TIMESTAMP,
    total_sent      INTEGER DEFAULT 0,
    total_failed    INTEGER DEFAULT 0,
    batch_count     INTEGER DEFAULT 0,
    status          TEXT,            -- done | partial | failed
    report_json     TEXT             -- полный JSON-отчёт
);

-- Кто был в каждой рассылке
CREATE TABLE send_recipients (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    send_id    INTEGER REFERENCES send_history(id),
    contact_id INTEGER REFERENCES contacts(id),
    email      TEXT NOT NULL,
    status     TEXT DEFAULT 'sent'   -- sent | failed | bounced
);

-- Запуски Research
CREATE TABLE research_runs (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    started_at   TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP,
    config_json  TEXT,    -- JSON: segment, region, count, keywords
    status       TEXT,    -- running | done | failed
    found_count  INTEGER DEFAULT 0,
    log_text     TEXT
);

-- Настройки приложения (key-value)
CREATE TABLE settings (
    key   TEXT PRIMARY KEY,
    value TEXT
);
-- Ключи settings: smtp_host, smtp_port, smtp_user, smtp_pass,
--                 from_name, from_email, reply_to,
--                 asset_url_mitino, asset_url_grekova,
--                 unsubscribe_url, app_login, app_password_hash
```

## Модули и страницы

### 0. Login (`/login`)

**Что на странице:**
- Поле логин
- Поле пароль
- Кнопка «Войти»
- Ошибка при неверных данных

**Логика:** Flask session; логин/пароль хранятся в таблице `settings` (пароль — bcrypt hash). Все остальные страницы закрыты без сессии.

### 1. Dashboard (`/`) — Главная

**Что на странице:**
- Карточки со статистикой:
  - Всего контактов в базе
  - Отправлено рассылок (всего)
  - Контактов без рассылки (новые)
  - Последний запуск Research (дата + сколько найдено)
- Блок «Последние рассылки» (5 строк: дата, шаблон, кол-во получателей, статус)
- Блок «Последний Research» (дата, параметры, статус)
- Быстрые кнопки: «Запустить Research», «Новая рассылка»

---

### 2. Research (`/research`)

**Что на странице:**

**Форма настроек поиска:**
- **Количество компаний** — числовое поле (10 / 25 / 50 или произвольное)
- **Сегменты компаний** — мультивыбор чекбоксами:
  - Электроника и приборостроение
  - Медтех и фармацевтика
  - Робототехника и автоматизация
  - IT-производство и hardware
  - Лазерные и оптические технологии
  - Прочее light industrial
- **Регион** — выпадающий список: Москва / Московская область / Вся Россия
- **Доп. ключевые слова** — текстовое поле (необязательно)
- **Кнопка «Запустить поиск»** (блокируется во время выполнения)

**Блок выполнения:**
- Статус: Ожидание / Выполняется / Завершено / Ошибка
- **Живой лог** (SSE-стрим из subprocess) — прокручиваемая консоль
- Прогресс-бар (если скрипт возвращает прогресс)

**Результаты последнего запуска:**
- Таблица: компания, ЛПР, email, телефон, сегмент
- Кнопка «Добавить всё в базу»
- Кнопка «Экспорт XLSX»

### 3. Контакты (`/contacts`)

**Что на странице:**

**Фильтры (панель сверху):**
- Поиск по тексту (имя, компания, email)

- Фильтр по региону
- Фильтр по статусу (new / sent / unsubscribed)
- Фильтр по дате (от — до)

**Таблица контактов:**
- Колонки: ☑ | Компания | ЛПР | Должность | Email | Телефон | Сегмент | Статус | Найден

**Панель выбора (появляется при выборе):**

- Кнопка «Отправить Митино» (неактивна если N > 30)
- Кнопка «Отправить Грекова» (неактивна если N > 30)
- Кнопка «Снять выбор»

**Дополнительно:**
- Кнопка «Добавить контакт вручную» (модальное окно с формой)
- Кнопка «Импорт из XLSX/CSV»
- Кнопка «Экспорт выбранных в XLSX»

### 4. Рассылки (`/campaigns`)

**Что на странице:**

**Форма новой рассылки:**
- **Шаблон письма** — переключатель с предпросмотром:
  - Промтехнопарк Митино (тема, прехедер, превью письма)
  - Грекова, 5–7 (тема, прехедер, превью письма)
- **Получатели** — два способа (вкладки):
  - *Из базы*: выбранные на странице Контакты (передаются параметром)
  - *Ручной ввод*: текстовое поле, по одному email на строку
- **Счётчик получателей**: «N адресов» + предупреждение если > 30
- **Предпросмотр HTML** (iframe с готовым письмом)
- **Кнопка «Отправить»** (неактивна если 0 или > 30 адресов)

**История рассылок (таблица ниже):**
- Дата | Шаблон | Отправлено | Ошибок | Статус | Детали ↗
- При клике «Детали» — модальное окно со списком всех получателей и их статусом

## Этапы реализации

### Этап 1 — Скелет (делает ИИ самостоятельно)
1. Создать папку `webapp/` в `/Desktop/work/`
2. Настроить Flask: `app.py`, `requirements.txt`
3. Создать SQLite схему (`database.py`)
4. Реализовать авторизацию (`auth.py`)
5. Создать `base.html` с навигацией
6. Страница Login — полностью рабочая

### Этап 2 — Контакты (делает ИИ самостоятельно)
1. Импорт существующих контактов из `Research/output/found_contacts_memory.jsonl` → SQLite
2. Страница Contacts с таблицей, фильтрами, пагинацией
3. Ручное добавление контакта (модальное окно)
4. Экспорт в XLSX

### Этап 3 — Рассылки (делает ИИ самостоятельно)
1. Скопировать HTML-шаблоны в `webapp/email_templates/`
2. Реализовать `mailer.py` (на базе существующего скрипта из Грекова)
3. Страница Campaigns: выбор шаблона, ввод адресов, счётчик ≤ 30
4. Отправка + запись в `send_history` и `send_recipients`
5. История рассылок

### Этап 4 — Настройки (делает ИИ самостоятельно)
1. Страница Settings: SMTP-форма, сохранение в SQLite
2. Кнопка «Проверить соединение»
3. Смена пароля приложения

### Этап 5 — Research (требует участия пользователя — см. ниже)
1. Форма настроек поиска
2. Запуск `run_accelerated_outreach.py` через subprocess
3. Стриминг лога через SSE
4. Сохранение результатов в SQLite

### Этап 6 — Dashboard (делает ИИ самостоятельно)
1. Агрегация статистики из SQLite
2. Блоки последних активностей

## Правила и ограничения

1. **Максимум 30 BCC** за одну отправку — жёстко в коде (UI блокирует > 30)
2. **SMTP_PASS никогда не логируется** и не попадает в HTML/лог
3. **Все получатели — только BCC**, To = s.zharov@abcentrum.ru
4. **Батчи:** если выбрано ≤ 30 → один батч; больше 30 — кнопка заблокирована
5. **Пароль** в settings хранится как bcrypt-hash (кроме SMTP_PASS — он в .env или зашифрован в SQLite)
6. **Research не блокирует UI** — subprocess запускается в фоне, лог стримится через SSE
